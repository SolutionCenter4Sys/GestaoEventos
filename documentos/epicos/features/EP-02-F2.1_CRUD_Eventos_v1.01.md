## 2. Resumo

Sistema completo de criação, leitura, atualização e exclusão (CRUD) de eventos, incluindo criação automática a partir de solicitação aprovada ou criação manual pelo Marketing/Organizador, com todos os campos necessários para gestão operacional do evento.

---

## 3. Descrição

Esta feature cobre o **CRUD completo de eventos**:

- **Criar evento** a partir de solicitação aprovada (dados pré-preenchidos automaticamente) ou manualmente
- **Editar evento** (todos os campos: nome, tipo, data/hora, local, descrição, professor, vagas, conteúdo público)
- **Visualizar evento** (detalhes completos + estatísticas de inscrições/presença)
- **Duplicar evento** (copiar para facilitar criação de eventos recorrentes)
- **Arquivar/Excluir evento** (soft delete, preserva histórico)
- **Campos principais**: nome, tipo (Workshop/Mentoria), formato (Presencial/Online/Híbrido), data/hora início/fim, local, descrição, professor designado, vagas máximas, conteúdo público, status do evento

A feature é base para todo o gerenciamento de eventos. Pré-requisito para F2.2, F2.3, F2.4 e EP-03 (inscrições).

---

## 4. Critérios de Aceite

- [ ] Marketing consegue criar evento manualmente com todos os campos
- [ ] Evento pode ser criado automaticamente a partir de solicitação aprovada (campos herdados)
- [ ] Todos os campos do evento podem ser editados (exceto ID e data de criação)
- [ ] Campos obrigatórios são validados (nome, tipo, formato, data/hora, vagas)
- [ ] É possível duplicar evento existente (cópia com nome modificado)
- [ ] Exclusão de evento é soft delete (preserva no BD, oculta da UI)
- [ ] Professor designado recebe notificação quando vinculado a evento
- [ ] Validação impede data/hora de término anterior à data/hora de início
- [ ] Upload de imagem de capa do evento funciona (opcional)
- [ ] Histórico de alterações do evento é registrado (auditoria)
