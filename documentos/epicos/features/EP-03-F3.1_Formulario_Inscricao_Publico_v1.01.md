## 2. Resumo

Formulário público web responsivo que permite participantes se inscreverem em eventos publicados, com campos obrigatórios (nome, e-mail, telefone, CPF), validações em tempo real, verificação de vagas disponíveis, confirmação automática por e-mail e criação de conta (login) para acompanhamento.

---

## 3. Descrição

Esta feature cobre o **formulário público de inscrição**:

- **Formulário web** acessível via link único do evento (ex: /eventos/123/inscricao)
- **Campos obrigatórios**: Nome completo, E-mail, Telefone, CPF
- **Validações em tempo real**: formato de e-mail, telefone (máscara), CPF (dígitos verificadores)
- **Verificação de vagas**: antes de permitir inscrição, valida se há vagas disponíveis
- **Unicidade**: um CPF pode se inscrever apenas uma vez por evento (validação)
- **Confirmação automática**: e-mail enviado imediatamente após inscrição com detalhes do evento + QR Code de check-in
- **Criação de conta**: ao se inscrever pela primeira vez, participante recebe senha para acessar área logada
- **Proteção anti-spam**: CAPTCHA ou honeypot para evitar bots
- **Responsividade**: interface adaptável a mobile, tablet, desktop

A feature é interface pública principal da plataforma. Depende de EP-02-F2.1 (Eventos) e EP-02-F2.2 (Capacidade). Base para F3.2 (Área do Participante).

## 4. Critérios de Aceite

- [ ] Formulário é acessível publicamente via link único do evento
- [ ] Campos obrigatórios são validados (nome, e-mail, telefone, CPF)
- [ ] Validação de formato de e-mail funciona corretamente
- [ ] Máscara de telefone é aplicada automaticamente ((XX) XXXXX-XXXX)
- [ ] Validação de CPF verifica dígitos verificadores
- [ ] Sistema impede inscrição duplicada (mesmo CPF no mesmo evento)
- [ ] Sistema valida se há vagas disponíveis antes de permitir inscrição
- [ ] Mensagem de erro clara é exibida se evento estiver esgotado
- [ ] E-mail de confirmação é enviado em até 1 minuto após inscrição
- [ ] E-mail contém: confirmação, detalhes do evento, QR Code de check-in, link para área do participante
- [ ] Participante recebe senha para criar conta na primeira inscrição
- [ ] CAPTCHA ou honeypot protege contra bots
- [ ] Formulário é totalmente responsivo (mobile, tablet, desktop)

## 6. Dependências

- **Épico:** EP-03
- **Features:** 
  - Depende de EP-02-F2.1 (CRUD Eventos) - eventos devem estar publicados
  - Depende de EP-02-F2.2 (Capacidade) - verificação de vagas
  - Depende de EP-02-F2.4 (Check-in) - geração de QR Code
  - Depende de EP-06-F6.2 (Gatilhos) - e-mail de confirmação
  - Depende de EP-08-F8.1 (Autenticação) - criação de conta
  - Base para F3.2 (Área do Participante)
- **Pré-condição:** Serviço de e-mail configurado, CAPTCHA (Google reCAPTCHA ou alternativa)

---

## 7. Observações

- Validação de CPF deve usar algoritmo correto de dígitos verificadores (módulo 11)
- Considerar campo "Como ficou sabendo do evento?" para análise de marketing (opcional)
- Senha gerada deve ser forte (8+ caracteres, maiúsculas, minúsculas, números)
- Participante deve poder alterar senha no primeiro acesso
- Taxa de conversão (acessos / inscrições) é métrica importante
