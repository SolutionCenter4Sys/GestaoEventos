## 2. Resumo

Portal logado exclusivo para participantes visualizarem seus dados cadastrais, eventos em que estão inscritos (confirmados, aguardando vaga, concluídos), status de presença, certificados disponíveis para download e possibilidade de atualizar dados pessoais.

---

## 3. Descrição

Esta feature cobre a **área logada do participante**:

- **Login** com e-mail e senha
- **Dashboard pessoal** com resumo: próximos eventos, certificados disponíveis, lista de espera
- **Meus Eventos** - lista de eventos (abas: Próximos, Concluídos, Lista de Espera)
- **Meus Certificados** - lista de certificados disponíveis com botão de download (PDF)
- **Meus Dados** - visualização e edição de dados cadastrais (nome, telefone, senha)
- **QR Codes** - acesso aos QR Codes de check-in dos próximos eventos
- **Histórico completo** de participação em eventos

A feature cria engajamento contínuo do participante com a plataforma. Depende de F3.1 (Inscrição) e EP-08 (Autenticação). Integra com EP-04 (Certificados).

---

## 4. Critérios de Aceite

- [ ] Participante consegue fazer login com e-mail e senha
- [ ] Dashboard exibe resumo: X próximos eventos, Y certificados disponíveis
- [ ] Aba "Próximos Eventos" lista eventos futuros com: nome, data, local, status (confirmado/lista espera), botão "Ver QR Code"
- [ ] Aba "Concluídos" lista eventos passados com: nome, data, presença registrada (sim/não), certificado disponível
- [ ] Aba "Lista de Espera" mostra eventos em que está na fila + posição atual
- [ ] Seção "Certificados" lista todos os certificados disponíveis com botão "Baixar PDF"
- [ ] Seção "Meus Dados" permite visualizar e editar: nome, telefone, e-mail (read-only), senha (trocar)
- [ ] Participante pode cancelar inscrição de eventos futuros (com confirmação)
- [ ] Participante pode sair voluntariamente de lista de espera
- [ ] Interface é responsiva (mobile, tablet, desktop)

## 6. Dependências

- **Épico:** EP-03
- **Features:** 
  - Depende de F3.1 (Inscrição) - participante deve ter inscrição
  - Depende de EP-04 (Certificados) - integração de certificados
  - Depende de EP-08-F8.1 (Autenticação) - login
- **Pré-condição:** Sistema de autenticação JWT ou session-based

---

## 7. Observações

- Área do participante deve ser mobile-first (muitos acessarão via smartphone)
- Notificações in-app (ex: novo certificado disponível) melhoram engajamento
- Considerar funcionalidade de compartilhamento de certificado em redes sociais
- Histórico completo de participação incentiva participação recorrente
- Link de "Recuperar senha" é essencial
